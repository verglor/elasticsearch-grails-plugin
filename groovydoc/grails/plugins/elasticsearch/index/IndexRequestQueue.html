Package: grails.plugins.elasticsearch.index

[Groovy] Class IndexRequestQueue

    • grails.plugins.elasticsearch.index.IndexRequestQueue

    • Holds objects to be indexed.
      It looks like we need to keep object references in memory until they indexed properly. If indexing fails, all failed objects are retried. Still no support for max number of retries (todo) NOTE: if cluster state is RED, everything will probably fail and keep retrying forever. NOTE: This is shared class, so need to be thread-safe.
